# 第 11 章：品質管控與持續優化

> 「AI 不會犯懶，但它會犯錯。你的工作是確保錯誤不會到達客戶手上。」

---

## 11.1 建立品質檢查機制

### 核心觀念
- AI 輸出不等於最終成品，品質檢查是不可省略的環節

### 內容要點
- **AI 常見的三大品質問題**：
  1. **幻覺（Hallucination）**：AI 編造不存在的事實或數據
     - 辨識方法：數據要求提供來源、事實要交叉驗證
     - 高風險場景：引用數據、歷史事件、法律條文、人物經歷
  2. **偏見（Bias）**：AI 的輸出可能帶有訓練數據的偏見
     - 辨識方法：注意刻板印象、單一觀點、文化偏見
     - 高風險場景：人物描寫、市場分析、群體描述
  3. **過時資訊（Outdated Information）**：AI 的知識有截止日期
     - 辨識方法：涉及最新資訊時要額外驗證
     - 高風險場景：技術工具推薦、市場數據、法規政策

- **品質檢查清單（Checklist）**：
  | 檢查項目 | 說明 | 優先級 |
  |---------|------|--------|
  | 事實正確性 | 數據、引用、名稱是否正確 | 高 |
  | 時效性 | 資訊是否過時 | 高 |
  | 邏輯一致性 | 前後論述是否矛盾 | 高 |
  | 完整性 | 是否回答了所有問題 | 中 |
  | 適當性 | 語氣、用詞是否適合目標讀者 | 中 |
  | 格式正確 | 是否符合要求的輸出格式 | 低 |

## 11.2 Agent 輸出的人工審核流程

### 核心觀念
- 不同類型的輸出需要不同程度的人工審核

### 內容要點
- **三級審核制度**：

  #### 第一級：快速掃描（適用於低風險內容）
  - 適用：內部筆記、初步草稿、創意發想
  - 審核時間：1-2 分鐘
  - 重點：大方向正確、無明顯錯誤

  #### 第二級：仔細審查（適用於中風險內容）
  - 適用：客戶文件、社群貼文、報告初稿
  - 審核時間：5-10 分鐘
  - 重點：事實驗證、語氣適當、品牌一致性

  #### 第三級：深度審核（適用於高風險內容）
  - 適用：法律文件、財務報告、公開發表的內容
  - 審核時間：15-30 分鐘
  - 重點：逐字確認、交叉驗證、專業人士複審

- **建立審核 SOP**：
  1. 識別內容風險等級
  2. 選擇對應的審核層級
  3. 按清單逐項檢查
  4. 記錄發現的問題
  5. 回饋給 Agent 修正（或人工修正）
  6. 存檔

## 11.3 效能指標與持續改善

### 核心觀念
- 沒有衡量就沒有改善——建立簡單的效能追蹤系統

### 內容要點
- **四個核心效能指標**：
  1. **時間節省率**：AI 協助前後的時間對比
     - 公式：(原本時間 - AI 協助後時間) / 原本時間 × 100%
  2. **品質合格率**：一次通過審核的比例
     - 目標：隨著使用優化，合格率應持續提升
  3. **修改次數**：平均需要幾輪修改才能達到可用品質
     - 目標：從 3-4 輪降到 1-2 輪
  4. **成本效益比**：AI 工具成本 vs. 產出價值
     - 計算方式：月訂閱費用 / 節省的工時（以時薪計算）

- **簡易追蹤方法**：
  - 用 Google Sheets 建立簡單的追蹤表
  - 每週花 10 分鐘回顧
  - 每月做一次效能報告

- **持續改善的 PDCA 循環**：
  1. **P**lan：規劃要改善的 Agent 或流程
  2. **D**o：實施改善（調整提示詞、換工具、修改流程）
  3. **C**heck：檢查改善效果（對比效能指標）
  4. **A**ct：如果有效就標準化，如果無效就換方法

## 11.4 常見問題與解決方案

### 內容要點
- **FAQ 整理**：

| 問題 | 原因 | 解決方案 |
|------|------|---------|
| AI 一直給錯誤的資訊 | 提示詞不夠具體 / 問題超出 AI 能力 | 增加上下文、交叉驗證、換工具 |
| AI 的回覆太長 / 太短 | 沒有指定長度要求 | 明確指定字數或段落數 |
| AI 的語氣不對 | 沒有設定角色和語氣 | 用 RCOAT 框架重新設定 |
| AI 不記得之前說過的話 | 對話上下文有限 | 定期摘要、使用 Projects 功能 |
| AI 的創意不夠 | 提示詞太限制 | 用「先發散再收斂」的策略 |
| 流程太複雜跑不通 | 一次自動化太多環節 | 簡化流程，逐步自動化 |
| 不知道哪個 AI 工具最好 | 工具太多 | 先專注用好 1-2 個主力工具 |

- **疑難排解決策樹**：
  - 提供一個簡單的流程圖，幫讀者快速定位問題和找到解法
