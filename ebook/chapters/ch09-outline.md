# 第 9 章：Agent Team 的協作與串聯

> 「一個 Agent 很強，一組 Agent 串起來就是一條生產線。」

---

## 9.1 建立 Agent 之間的工作流程

### 核心觀念
- 單獨使用 Agent 是「單點作業」，串聯 Agent 才是「系統化運作」
- 關鍵：定義清楚的輸入/輸出介面

### 內容要點
- **串聯的三個前提**：
  1. 每個 Agent 的輸入/輸出格式要標準化
  2. 定義清楚的「交接點」（誰的輸出接到誰的輸入）
  3. 設置品質檢查點（不是所有步驟都自動跑完）

- **常見的串聯模式**：
  1. **線性串聯**：A → B → C（內容製作最常用）
  2. **分支匯聚**：A → [B, C, D] → E（研究調查最常用）
  3. **循環迭代**：A → B → 審核 → (不通過) → A（品質提升最常用）
  4. **條件分支**：A → 判斷 → B 或 C（客服分流最常用）

- **標準化模板設計**：
  - 輸入模板：任務描述、上下文、參考資料、輸出格式要求
  - 輸出模板：標題、正文、摘要、後續建議、品質自評

## 9.2 任務分配與自動化串接

### 核心觀念
- 從「手動複製貼上」到「自動化流程」的進化

### 內容要點
- **手動串接法**（入門）：
  - 人工將 Agent A 的輸出複製到 Agent B 的輸入
  - 適合剛開始建立流程、還在測試的階段
  - 優點：完全掌控；缺點：耗時

- **半自動串接法**（進階）：
  - 用 AI 平台的 Projects / GPTs 預設好角色和上下文
  - 建立模板化的提示詞，減少每次的設定時間
  - 用文件作為「共享記憶」傳遞資訊

- **全自動串接法**（高階）：
  - 用自動化工具串聯多個 AI 步驟
  - 觸發條件 → 自動執行 → 自動交付
  - 下一節詳細介紹工具

## 9.3 使用 Zapier / Make / n8n 串聯 Agent

### 核心觀念
- 自動化工具是「Agent 之間的橋樑」

### 內容要點

#### Zapier（入門友善）
- 優勢：介面簡單、整合平台最多（6,000+）
- 限制：免費額度有限、複雜流程較貴
- 範例流程：Gmail 新信 → ChatGPT 分類 → Slack 通知

#### Make（視覺化流程）
- 優勢：可視化拖拉介面、複雜流程支援好
- 限制：學習曲線稍高
- 範例流程：表單提交 → AI 分析 → 自動回覆 + 建立待辦

#### n8n（技術人員首選）
- 優勢：開源免費、自架掌控、無限制
- 限制：需要基本技術能力
- 範例流程：RSS 新聞 → AI 摘要 → 自動發布社群

#### 工具選擇指引
| 適合對象 | 建議工具 | 原因 |
|---------|---------|------|
| 完全新手 | Zapier | 最簡單，模板多 |
| 進階使用者 | Make | 彈性高，性價比好 |
| 技術人員 | n8n | 免費開源，完全掌控 |

## 9.4 打造端到端的自動化工作流

### 核心觀念
- 終極目標：建立「一鍵啟動」的完整工作流

### 內容要點
- **自動化工作流設計五步驟**：
  1. 畫出流程圖（先在紙上規劃）
  2. 手動測試流程（確認每一步都跑得通）
  3. 逐步自動化（一次自動化一個環節）
  4. 設置例外處理（錯誤時怎麼辦？）
  5. 持續監控與優化

### 完整案例
- **情境**：自動化的「客戶開發流程」

```
觸發：收到新客戶詢問（Email / 表單）
  ↓
Step 1：AI 分類 → 識別客戶類型與需求
  ↓
Step 2：AI 研究 → 查詢客戶背景資訊
  ↓
Step 3：AI 撰寫 → 個人化的初步回覆
  ↓
Step 4：人類審核 → 確認後自動發送
  ↓
Step 5：AI 追蹤 → 3 天未回覆自動提醒
  ↓
Step 6：AI 紀錄 → 更新 CRM 資料庫
```
