# 第 2 章：認識你的 AI Agent 團隊

> 「當你學會把 ChatGPT 從一個萬事通變成一組專業團隊，一切就不一樣了。」

---

## 2.1 什麼是 AI Agent？（與傳統 AI 工具的差別）

### 核心觀念
- **AI 工具**：你給指令，它給結果（一問一答）
- **AI Agent**：你給目標，它規劃步驟、自主執行、回報結果

### 內容要點
- 對比表格：

| 維度 | AI 工具（Tool） | AI Agent（代理人） |
|------|----------------|-------------------|
| 互動模式 | 一問一答 | 持續對話、主動追問 |
| 任務範圍 | 單一任務 | 複合任務、多步驟 |
| 記憶能力 | 無 / 有限 | 有上下文記憶 |
| 角色設定 | 通用 | 專職角色、特定專長 |
| 自主性 | 被動回應 | 主動規劃、執行 |

- 舉例說明：
  - Tool 模式：「幫我寫一篇 500 字的產品介紹」→ 得到一篇文章
  - Agent 模式：「你是我的行銷文案專員，負責我們所有產品的文案。先了解我們的品牌調性，再幫我寫產品介紹，寫完後自我檢查是否符合品牌風格。」

### 建議案例
- 同一個任務「寫週報」，用 Tool 模式 vs. Agent 模式的差異展示

## 2.2 Agent 的核心能力：理解、推理、執行、學習

### 核心觀念
- 用四個核心能力來說明 Agent 為什麼「像一個團隊成員」

### 內容要點

#### 理解力（Understanding）
- 理解自然語言指令、上下文、隱含需求
- 比喻：像一個聽得懂「言外之意」的資深同事

#### 推理力（Reasoning）
- 拆解複雜問題、規劃執行步驟、做出判斷
- 比喻：像一個會「先想再做」的專案經理

#### 執行力（Execution）
- 生成文本、分析資料、撰寫程式碼、搜尋資訊
- 比喻：像一個手腳俐落的執行專員

#### 學習力（Learning）
- 透過上下文累積經驗、根據回饋調整輸出
- 比喻：像一個「越用越順手」的合作夥伴
- 注意：目前的 AI 學習是在對話範圍內，不是永久記憶（管理讀者預期）

## 2.3 為什麼需要「團隊」而不是「單一工具」？

### 核心觀念
- 「萬能 Agent」的迷思：一個 Agent 做所有事 → 品質不穩定
- 「專職 Agent」的優勢：每個 Agent 專注一個角色 → 品質可控

### 內容要點
- **問題說明**：為什麼同一個 AI 既當寫手又當分析師效果不好？
  1. 角色混淆：同時要「有創意」又要「嚴謹分析」是矛盾的
  2. 上下文汙染：前一個任務的風格會影響下一個任務
  3. 品質不穩：沒有專門的審核機制

- **團隊分工的優勢**：
  1. 專業分工：每個 Agent 有清楚的角色和專長
  2. 品質保證：寫作 Agent 寫、編輯 Agent 改、品管 Agent 審
  3. 效率提升：多個 Agent 可以平行工作
  4. 可重現性：流程標準化，品質穩定

### 建議案例
- **對比實驗**：同一份行銷提案
  - 方法 A：用一個 AI 從頭到尾做完
  - 方法 B：用 Agent Team（策略 Agent → 寫作 Agent → 編輯 Agent → 設計 Agent）
  - 比較兩者的品質、效率、可調整性

## 2.4 Agent Team 的協作模式

### 核心觀念
- 介紹三種基本的 Agent 協作模式

### 內容要點

#### 模式一：流水線模式（Sequential）
```
Agent A → Agent B → Agent C → 成品
（寫作）   （編輯）   （排版）
```
- 適用場景：內容生產、報告撰寫
- 優點：品質逐步提升
- 缺點：速度較慢

#### 模式二：平行處理模式（Parallel）
```
         ┌→ Agent A（市場分析）──┐
任務拆解 ─┼→ Agent B（競品分析）──┼→ 彙整
         └→ Agent C（客戶訪談）──┘
```
- 適用場景：研究調查、資料蒐集
- 優點：速度快
- 缺點：需要最後彙整

#### 模式三：混合模式（Hybrid）
- 結合流水線與平行處理
- 適用場景：大型專案
- 這是 AI 智能工作術最推薦的模式

### 建議案例
- 用「製作一份營運月報」的完整流程，展示混合模式的運作
