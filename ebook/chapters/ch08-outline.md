# 第 8 章：程式開發 Agent — 你的技術部門

> 「不會寫程式？沒關係。AI 讓每個人都能擁有自己的技術團隊。」

---

## 8.1 程式碼生成 Agent：從需求到程式碼

### 核心觀念
- AI 程式碼生成不是魔法，是「把你的想法翻譯成程式語言」
- 不會寫程式的人：可以用 AI 做出簡單的工具和自動化
- 會寫程式的人：可以用 AI 大幅加速開發流程

### 內容要點
- **不會寫程式的人能做什麼**：
  1. Google Sheets / Excel 自動化公式與巨集
  2. 簡單的網頁表單與著陸頁
  3. Zapier / Make 的自動化流程
  4. 用 No-code 平台 + AI 輔助建立應用

- **會寫程式的人如何用 AI 加速**：
  1. 需求 → 程式碼框架（Scaffolding）
  2. 自動完成重複性程式碼
  3. 技術方案設計與比較
  4. Debug 與問題排除

- **設定程式碼 Agent 的技巧**：
  - 指定程式語言和框架
  - 提供現有程式碼的上下文
  - 要求加上註解說明
  - 要求遵循特定的程式碼風格

### 工具推薦
- Claude Code / GitHub Copilot / Cursor
- ChatGPT Code Interpreter
- Replit AI

## 8.2 測試 Agent：自動化測試與除錯

### 核心觀念
- 寫完程式碼只是開始，確保它正確運作才是關鍵

### 內容要點
- **測試 Agent 的三種用法**：
  1. **程式碼審查**：請 AI 檢查程式碼中的潛在問題
  2. **測試案例生成**：根據功能需求自動產生測試案例
  3. **Bug 排除**：貼上錯誤訊息，讓 AI 分析原因和解法

- **非技術人員也能用的除錯方法**：
  - 把錯誤訊息貼給 AI，用白話描述「我按了什麼然後怎樣了」
  - AI 會引導你一步步排除問題

- **實用場景**：
  - Excel 巨集跑不動 → AI 幫你找問題
  - 網站某個功能壞了 → AI 幫你定位問題
  - 自動化流程突然停了 → AI 幫你排除

## 8.3 部署 Agent：自動化部署流程

### 核心觀念
- 「做完」和「上線」之間，部署 Agent 幫你銜接最後一哩路

### 內容要點
- **簡易部署方案（非技術人員）**：
  1. GitHub Pages（免費靜態網站託管）
  2. Vercel / Netlify（一鍵部署）
  3. Notion / Carrd（超簡易建站）

- **用 AI 輔助部署的方法**：
  - 把部署文件的步驟貼給 AI，請它轉化為你能理解的操作指引
  - 遇到錯誤時，截圖或複製錯誤訊息讓 AI 幫你解決

- **自動化部署（技術人員）**：
  - CI/CD 流程設計
  - 用 AI 撰寫部署腳本
  - 監控與告警設定

## 8.4 實戰演練：用開發 Agent 團隊建立一個網頁應用

### 完整案例（分兩條路線）

#### 路線 A：非技術人員版
- **任務**：用 AI + No-code 工具建立一個「客戶回饋收集表單」
- **步驟**：
  1. 用 AI 設計表單的欄位與邏輯
  2. 用 Google Forms / Typeform 建立表單
  3. 用 Zapier + AI 自動分析回饋內容
  4. 用 AI 生成每週回饋摘要報告

#### 路線 B：技術人員版
- **任務**：用 AI Agent 團隊開發一個「個人品牌網站」
- **Agent Team 分工**：

```
需求 Agent → 釐清網站功能需求
    ↓
架構 Agent → 設計技術方案（HTML/CSS/JS 或框架選擇）
    ↓
開發 Agent → 生成前端程式碼
    ↓
測試 Agent → 程式碼審查 + 跨瀏覽器測試
    ↓
部署 Agent → 部署到 GitHub Pages / Vercel
    ↓
人類審核   → 確認功能與視覺效果
```
