## 3.1 工作術的三層架構：策略層、執行層、品管層

### 核心觀念
- 任何有效的工作系統都需要三個層次
- 這三層對應到傳統公司的：管理層 → 執行團隊 → 品管部門

### 內容要點

#### 策略層（Strategy Layer）
- **負責人**：人類（你）+ 策略 Agent
- **職責**：
  - 定義目標與方向
  - 拆解任務與分配
  - 資源配置與優先順序
- **類比**：公司的 CEO + 策略顧問

#### 執行層（Execution Layer）
- **負責人**：各專職 Agent
- **職責**：
  - 依照指令執行具體任務
  - 回報進度與問題
  - 產出初步成果
- **類比**：公司各部門的執行團隊

#### 品管層（Quality Control Layer）
- **負責人**：品管 Agent + 人類最終審核
- **職責**：
  - 檢查輸出品質
  - 確認是否符合目標
  - 回饋改善建議
- **類比**：公司的 QA 部門

## 3.2 人類的角色：總指揮與決策者

### 核心觀念
- AI 再強大，仍然需要人類做最關鍵的事
- 人類的不可取代價值在於：創意判斷、價值觀、情感連結、責任承擔

### 內容要點
- **人類應該做的事**：
  1. 設定目標與願景（AI 不知道你要什麼）
  2. 做價值判斷（哪個方案更好取決於你的價值觀）
  3. 建立人際關係（客戶信任的是你，不是 AI）
  4. 承擔最終責任（AI 的輸出由你負責）
  5. 發現新機會（AI 擅長執行，但創新來自人類洞察）

- **人類不應該浪費時間做的事**：
  1. 重複性的文書工作
  2. 資料彙整與格式化
  3. 初稿撰寫（尤其是制式文件）
  4. 資料搜尋與整理
  5. 排程與基本計算

### 建議案例
- **反面案例**：完全依賴 AI 不審核，導致錯誤的真實案例
- **正面案例**：人機協作，人類負責審核和決策，效率與品質兼顧

## 3.3 工作流程設計的基本原則

### 核心觀念
- 好的工作流程 = 清楚的輸入 + 明確的步驟 + 可預期的輸出
- 五個設計原則

### 內容要點

#### 原則一：任務明確化
- 每個 Agent 的任務要具體、可衡量
- 壞的任務：「幫我做行銷」
- 好的任務：「分析過去 3 個月的 Facebook 貼文數據，找出互動率最高的前 5 篇，分析共同特徵」

#### 原則二：單一職責
- 每個 Agent 只負責一件事
- 不要讓寫作 Agent 同時校對（那是編輯 Agent 的事）

#### 原則三：標準化輸入/輸出
- 定義每個環節的輸入格式和輸出格式
- 例如：寫作 Agent 的輸出格式 = Markdown、含標題層級、字數標註

#### 原則四：設置檢查點
- 不是所有步驟都自動跑完，關鍵節點要人類檢查
- 建議檢查點：策略確認後、初稿完成後、最終發佈前

#### 原則五：迭代改善
- 第一次的流程不會完美，重點是持續優化
- 記錄每次的問題與改善方案

## 3.4 從需求到成果的完整流程

### 核心觀念
- 展示一個完整的「端到端」工作流程範例
- 讓讀者看到全貌，建立信心

### 內容要點
- 六步驟標準流程：

```
步驟 1：定義需求
  └─ 你想要什麼？給誰看？達成什麼目標？

步驟 2：任務拆解
  └─ 把大任務拆成小任務，決定哪些交給哪個 Agent

步驟 3：角色指派
  └─ 為每個 Agent 設定角色、提供上下文、給予明確指令

步驟 4：執行與監控
  └─ Agent 開始工作，你監控進度、回答 Agent 的問題

步驟 5：審核與迭代
  └─ 檢查輸出品質、提供修改回饋、Agent 修正

步驟 6：交付與歸檔
  └─ 確認最終成品、歸檔流程（方便下次重複使用）
```

### 完整案例演練
- **任務**：為一個新產品準備上市簡報
- 逐步展示六個步驟的實際操作：
  1. 需求定義：「為新推出的線上課程做一份 10 頁的上市簡報」
  2. 任務拆解：市場定位 / 目標客群分析 / 內容大綱 / 文案撰寫 / 視覺設計
  3. 角色指派：策略 Agent → 寫作 Agent → 視覺 Agent
  4. 執行過程：展示每個 Agent 的提示詞與輸出
  5. 審核修改：指出初稿的問題，展示修改指令
  6. 最終成品：展示完成的簡報架構
