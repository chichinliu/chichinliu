# 第 4 章：策略規劃 Agent — 你的智囊團

> 「好的決策不是靠直覺，而是靠充分的資訊與系統化的分析。」

你有沒有這樣的經驗？

接到一個新專案，腦子裡有一堆想法，但坐到電腦前，卻不知道從哪裡開始。該先做市場調查嗎？該先寫企劃書嗎？該先估預算嗎？然後你打開 Google，搜尋了兩個小時，開了十幾個分頁，最後發現自己還是在原地打轉。

這不是你的問題。這是「一個人要同時當策略長、分析師和企劃」的問題。

在傳統工作模式裡，策略規劃是最耗腦力的環節。你需要蒐集大量資訊、分析各種可能性、評估風險和回報，最後做出決策。大公司有整個策略部門來做這件事，但如果你是自由工作者、小型團隊或一人公司呢？

現在，你可以組建自己的「AI 智囊團」。

這一章，我會教你建立三個策略規劃 Agent——市場分析 Agent、企劃 Agent、決策輔助 Agent。他們各有專長，協同運作，幫你從「模糊的想法」走到「清晰的行動計畫」。

## 4.1 市場分析 Agent：你的專屬市場研究員

### 為什麼你需要市場分析 Agent？

過去，做一份像樣的市場分析報告，你需要：訂閱產業研究報告（一份動輒數萬元）、花好幾天蒐集和整理資料、再花幾天寫成報告。這是大企業的遊戲，小公司和個人工作者根本玩不起。

但 AI 改變了這個遊戲規則。

市場分析 Agent 不會取代專業的市場研究機構，但它能幫你在 30 分鐘內得到一份「80 分」的分析，讓你快速掌握市場全貌，做出更有根據的決策。

### 設定你的市場分析 Agent

打開你慣用的 AI 工具（ChatGPT、Claude 或其他），用以下的角色設定來建立你的市場分析 Agent：

```
【角色設定】

你是一位擁有 15 年經驗的市場研究分析師，專長包括：
- 產業趨勢分析與市場規模估算
- 競爭對手追蹤與競爭態勢分析
- 目標客群輪廓描繪與消費者洞察
- 商業機會識別與進入策略建議

你的分析風格是：
- 數據導向：盡可能提供具體數據佐證
- 結構清晰：使用框架和表格呈現分析結果
- 行動導向：每個分析都要附上可行動的建議
- 客觀平衡：同時呈現機會與風險

輸出格式要求：
- 使用 Markdown 格式
- 重要數據用粗體標示
- 分析結論用條列式呈現
- 每份報告結尾附上「行動建議」和「需要進一步驗證的資訊」
```

這段角色設定做了幾件重要的事：

1. **定義專業身分**：不是「通用 AI」，而是「資深市場分析師」，這會影響 AI 的回答風格和深度
2. **指定能力範圍**：讓 AI 知道你期待它做哪些類型的分析
3. **設定工作風格**：數據導向、結構清晰——這確保輸出的品質穩定
4. **規範輸出格式**：減少每次要求格式的溝通成本

### 四個實用的分析模板

有了角色設定，接下來你需要幾個常用的「任務模板」。就像你給員工的工作表單，有了模板，每次只需要填入不同的主題，就能快速啟動分析。

#### 模板一：SWOT 分析

```
請針對「[你的產品/服務/公司]」進行 SWOT 分析。

背景資訊：
- 產業：[填入產業]
- 目標市場：[填入市場]
- 主要產品/服務：[填入描述]
- 目前營運狀況：[填入概述]

請依照以下格式輸出：
1. Strengths（優勢）：列出 3-5 項，每項附上簡短說明
2. Weaknesses（劣勢）：列出 3-5 項，每項附上改善建議
3. Opportunities（機會）：列出 3-5 項，每項附上把握策略
4. Threats（威脅）：列出 3-5 項，每項附上因應對策
5. 策略建議：綜合以上分析，提出 3 個優先行動項目
```

#### 模板二：競爭對手分析

```
請幫我分析以下競爭對手：

我的產品/服務：[填入描述]
主要競爭對手：
1. [競爭對手 A]
2. [競爭對手 B]
3. [競爭對手 C]

請從以下維度進行比較分析：
- 產品/服務特色
- 定價策略
- 目標客群
- 行銷管道
- 品牌定位
- 優勢與弱點

輸出格式：
1. 競爭對手比較表格
2. 各競爭對手的詳細分析
3. 我的競爭優勢與差異化機會
4. 建議的競爭策略
```

#### 模板三：市場趨勢掃描

```
請幫我掃描「[填入產業/市場]」的最新市場趨勢。

我需要了解：
1. 市場規模與成長率（如果有公開數據）
2. 近 1-2 年的主要趨勢變化
3. 消費者行為的轉變
4. 新興技術或商業模式的影響
5. 法規或政策面的變化
6. 預期的未來發展方向

請特別標注哪些資訊需要我進一步用最新數據驗證。
```

#### 模板四：目標客群分析

```
請幫我建立「[產品/服務]」的目標客群輪廓。

產品/服務描述：[填入描述]
目前已知的客群特徵：[填入你知道的資訊]

請分析：
1. 主要目標客群（Primary）：人口統計、心理特徵、行為模式
2. 次要目標客群（Secondary）：同上
3. 每個客群的痛點與需求
4. 他們的購買決策流程
5. 觸及這些客群的最佳管道
6. 對他們有效的訊息框架
```

### 使用市場分析 Agent 的注意事項

市場分析 Agent 很強大，但你需要記住三件事：

**第一，AI 的資料有時效性限制。**

大部分 AI 模型的訓練資料都有截止日期。如果你需要最新的市場數據，應該搭配具有即時搜尋功能的工具（例如 Perplexity、ChatGPT 的網頁瀏覽功能），或者自己用 Google Trends、SimilarWeb 等工具做交叉驗證。

**第二，數據要交叉驗證，不可全盤接受。**

AI 有時會「看起來很有自信地」給你不正確的數據。這就是所謂的「幻覺」（Hallucination）。特別是具體的數字——市場規模、成長率、市佔率——一定要跟原始資料來源核對。一個好習慣是在提示詞中加上：「請標明資料來源，如果不確定某個數據的準確性，請明確告知。」

**第三，AI 擅長框架分析，但洞察仍需人類判斷。**

AI 可以幫你用 SWOT 框架整理資訊，但「這個機會值不值得追」、「這個威脅有多嚴重」——這些判斷需要你的產業經驗和商業直覺。把 AI 當成幫你整理牌桌的助手，出牌的決定還是你來做。

## 4.2 企劃 Agent：從想法到計畫的加速器

### 「想法」和「計畫」之間，差了什麼？

答案是：結構。

很多人有好想法，但遲遲無法執行，不是因為想法不好，而是因為沒有把想法轉化成具體、可執行的計畫。這個過程需要：定義清楚的目標、拆解成可執行的步驟、安排合理的時程、配置必要的資源。

這正是企劃 Agent 的強項。

### 設定你的企劃 Agent

```
【角色設定】

你是一位資深專案規劃顧問，擅長將模糊的想法轉化為具體可執行的計畫。

你的工作方式：
1. 先釐清目標：透過提問確認真正的目標（使用 SMART 原則）
2. 再設計結構：將大目標拆解為里程碑和工作項目
3. 最後輸出計畫：產生完整的專案計畫文件

你的核心能力：
- 目標設定與 SMART 化
- 工作分解結構（WBS）設計
- 里程碑與時程規劃
- 資源需求評估
- 風險預判與備案規劃

輸出格式要求：
- 使用 Markdown 格式
- 時程用表格呈現
- 重要里程碑用粗體標示
- 每個工作項目標注預估所需時間
- 計畫結尾附上「前三件要做的事」作為立即行動指引
```

### 企劃 Agent 的三步驟工作法

和企劃 Agent 協作，我推薦用「三步驟工作法」。不要一次把所有事情丟給它，分階段推進效果更好。

#### 第一步：目標釐清

先讓企劃 Agent 幫你把模糊的想法變成清晰的目標。

```
我有一個想法，但還不太明確，請幫我釐清。

我的想法是：[用一兩句話描述你的想法]

請用以下方式幫我釐清：
1. 先問我 5 個關鍵問題，幫助我定義更清楚的目標
2. 根據我的回答，用 SMART 原則幫我重新定義目標
   - Specific（具體的）
   - Measurable（可衡量的）
   - Achievable（可達成的）
   - Relevant（有關聯的）
   - Time-bound（有時限的）
```

這一步非常重要。很多專案失敗不是因為執行不力，而是因為一開始的目標就不清楚。讓 AI 當你的「釐清教練」，透過提問幫你想清楚。

#### 第二步：結構設計

目標確認後，讓企劃 Agent 幫你拆解成可執行的結構。

```
目標已確認：[貼上第一步確認的 SMART 目標]

請幫我設計執行計畫的結構：
1. 將目標拆解為 3-5 個主要里程碑
2. 每個里程碑再拆解為具體的工作項目
3. 為每個工作項目預估所需時間
4. 標出各項目之間的依賴關係（哪些要先做、哪些可以同時做）
5. 識別可能的風險點和備案
```

#### 第三步：計畫輸出

最後，讓企劃 Agent 產生一份完整的計畫文件。

```
請根據以上的結構，產生一份完整的專案計畫書，包含：

1. 專案概述（目標、範圍、預期成果）
2. 里程碑時程表（表格格式）
3. 詳細工作項目清單（含負責人欄位、預估時數）
4. 風險管理計畫（已識別的風險 + 因應對策）
5. 立即行動清單（接下來 7 天內要完成的前 3 件事）
```

### 三個高頻使用場景的模板

#### 場景一：專案啟動計畫

```
我即將啟動一個新專案，請幫我制定啟動計畫。

專案名稱：[填入]
專案目標：[填入]
預計時程：[填入，例如「3 個月內完成」]
可用資源：[填入，例如「1 人全職 + 每月 2 萬預算」]
已知限制：[填入，例如「沒有技術團隊」]

請提供完整的專案啟動計畫，包含階段規劃、里程碑、工作項目和風險評估。
```

#### 場景二：活動企劃

```
我需要規劃一個活動，請幫我制定企劃。

活動類型：[填入，例如「線上研討會」]
活動目的：[填入，例如「推廣新產品，目標 200 人報名」]
預計日期：[填入]
預算範圍：[填入]
目標參加者：[填入]

請提供完整的活動企劃，包含：活動流程、宣傳計畫、準備事項時程表、當天執行 checklist。
```

#### 場景三：產品規劃

```
我正在規劃一個新產品，請幫我制定產品開發計畫。

產品概念：[填入]
目標市場：[填入]
預期上線時間：[填入]
團隊規模：[填入]

請提供：產品開發路線圖、MVP 功能清單、開發階段規劃、上市前準備清單。
```

## 4.3 決策輔助 Agent：讓選擇不再困難

### 為什麼決策這麼難？

因為我們的大腦同時在處理太多變數。

「要不要接這個案子？」——你要考慮報酬、時間成本、學習價值、機會成本、客戶配合度......一下子有七八個因素要權衡，大腦很容易短路，最後靠「感覺」做決定。

感覺不是不好，但如果能先用系統化的方法把各個因素攤開來看清楚，你的「感覺」會做出更好的判斷。

這就是決策輔助 Agent 的價值。它不是替你做決定，而是幫你把決策所需的資訊整理清楚。

### 設定你的決策輔助 Agent

```
【角色設定】

你是一位企業策略顧問，專長是協助客戶做出理性、全面的決策。

你的分析方法：
- 不預設立場，客觀呈現各方案的優劣
- 使用結構化的分析框架，避免遺漏重要因素
- 量化可量化的因素，質化描述無法量化的因素
- 明確標示不確定性和假設條件

你的工作原則：
- 永遠不替客戶做決定，而是提供充分的分析依據
- 主動指出客戶可能忽略的盲點
- 如果某個方案有明顯的風險，要明確提醒

輸出格式：
- 分析報告使用表格和矩陣呈現
- 風險用「高/中/低」標示
- 最後提供「如果是我，我會考慮......」的顧問觀點（但強調最終決策由你做）
```

### 四種決策分析框架

不同類型的決策適合不同的分析框架。以下是四個最實用的，涵蓋了你日常會遇到的大部分決策場景。

#### 框架一：利弊分析矩陣

最簡單直覺的方法，適合快速評估。

```
我正在考慮 [決策描述]，有以下選項：
- 選項 A：[描述]
- 選項 B：[描述]
- 選項 C：[描述]（如果有的話）

我最在意的因素是：[列出，例如「成本、時間、品質、風險」]

請幫我做一份利弊分析矩陣：
1. 列出每個選項的優點（至少 3 個）
2. 列出每個選項的缺點（至少 3 個）
3. 針對我在意的因素逐一比較
4. 在最後給出你的顧問觀點
```

#### 框架二：加權評分矩陣

當你有多個評估標準，而且重要性不同時使用。

```
我需要從以下選項中做出選擇：
- 選項 A：[描述]
- 選項 B：[描述]
- 選項 C：[描述]

評估標準和權重（我已排好優先順序）：
1. [標準 1]：權重 [X]%
2. [標準 2]：權重 [X]%
3. [標準 3]：權重 [X]%
4. [標準 4]：權重 [X]%
5. [標準 5]：權重 [X]%
（權重加總 = 100%）

請用 1-10 分為每個選項在每個標準上評分，計算加權總分，並用表格呈現結果。最後解釋評分的依據。
```

#### 框架三：風險評估表

當決策涉及風險時，用這個框架看清楚「最壞的情況有多壞」。

```
我計畫 [描述你的計畫]。

請幫我做風險評估：
1. 列出可能面臨的所有風險（至少 5 個）
2. 每個風險評估：
   - 發生機率（高/中/低）
   - 影響程度（高/中/低）
   - 風險等級（機率 × 影響）
3. 每個風險提出因應對策
4. 標出「不可接受的風險」（需要在決策前先解決的）

用表格格式呈現，最後總結整體風險水平。
```

#### 框架四：情境分析

適合面對高度不確定性的決策——當你不確定未來會怎樣發展時。

```
我正在考慮 [決策描述]。

未來有幾種可能的發展情境，請幫我分析：

1. 樂觀情境：如果一切順利......
   - 這個決策會帶來什麼結果？
   - 預期回報是什麼？

2. 基準情境：如果發展正常......
   - 這個決策會帶來什麼結果？
   - 預期回報是什麼？

3. 悲觀情境：如果遇到困難......
   - 這個決策會帶來什麼結果？
   - 最壞的損失是什麼？
   - 是否有退出策略？

請針對每個情境給出具體的分析，並評估「做 vs. 不做」的整體風險回報比。
```

### 決策輔助 Agent 的最佳實踐

**先自己想過，再問 AI。**

不要一遇到決策就直接丟給 AI。先花 10 分鐘自己想一想，列出你已經考慮到的因素和初步傾向。然後再把這些告訴 AI，請它幫你檢查是否有遺漏。這樣做有兩個好處：一是你的思考不會被 AI 的分析「帶著走」，二是 AI 能根據你的初步想法提供更針對性的補充。

**問 AI「我可能忽略了什麼」。**

這是一個非常有力的提問。在 AI 給你分析報告後，追問一句：「根據以上分析，你覺得我可能忽略了哪些重要因素？有沒有什麼盲點我應該注意？」AI 通常會給你一些你確實沒想到的觀點。

**最終決策永遠是你的。**

AI 能幫你整理思緒、分析利弊、量化比較，但按下「確定」按鈕的是你。因為只有你知道自己的價值觀、偏好和不能妥協的底線。好的決策工具幫你看清全局，但不替你選擇方向。

## 4.4 實戰演練：用策略 Agent 團隊規劃一門線上課程

光看理論不夠，讓我們走一遍完整的實戰流程。

### 情境設定

你是一位自由工作者，專長是社群行銷。你想把自己的經驗整理成一門線上課程來銷售，但你從來沒做過線上課程，不知道該從哪裡開始。

我們會依序使用三個策略 Agent，一步步從「想法」走到「可執行的計畫」。

### 第一步：用市場分析 Agent 做市場調查

首先，你需要了解線上課程市場的現況。

**你的提示詞：**

```
我是一位社群行銷的自由工作者，正在考慮推出一門「社群行銷」
主題的線上課程。

請幫我做市場分析：
1. 台灣線上課程市場的概況和近期趨勢
2. 「社群行銷」類課程的競爭狀況
   （請分析 Hahow、PressPlay、Udemy 等平台上的類似課程）
3. 目標學員可能的輪廓和需求
4. 目前市場上的缺口或機會

請特別標注哪些資訊需要我自行驗證。
```

**AI 的回應（摘要）：**

> **市場概況**
> 台灣線上課程市場持續成長，疫情後線上學習已成為常態。根據公開資訊，Hahow 累積學員數已突破百萬，顯示市場接受度高。行銷類課程是平台上的熱門類別之一。
>
> **競爭分析**
> 目前各平台上已有多門社群行銷相關課程，主要集中在：
> - Facebook / Instagram 廣告投放（最多）
> - 社群經營基礎概念
> - 內容行銷策略
>
> **市場缺口**（潛在機會）
> - 大多數課程偏向「單一平台」教學，缺乏「跨平台整合策略」
> - 針對「一人公司 / 小型團隊」的實戰課程較少
> - 結合 AI 工具的社群行銷課程幾乎沒有 ← **這可能是你的機會**
>
> **目標學員輪廓**
> - 25-40 歲的自由工作者、小型企業主、行銷新手
> - 痛點：預算有限、沒有團隊、需要快速看到效果
>
> ⚠️ *需驗證：具體的市場規模數字和各平台的最新課程數量，建議直接上各平台查看。*

**你的審核與判斷：**

你上 Hahow 和 PressPlay 實際搜尋「社群行銷」，確認 AI 的分析大致正確。確實，結合 AI 工具的社群行銷課程很少——這個方向值得深入。你決定把課程定位調整為：「用 AI 工具 10 倍速做社群行銷」。

### 第二步：用企劃 Agent 設計課程規劃

方向確定後，用企劃 Agent 把它變成具體計畫。

**你的提示詞：**

```
我要開設一門線上課程，已完成初步市場分析，現在需要制定課程企劃。

課程定位：「用 AI 工具 10 倍速做社群行銷」
目標學員：自由工作者和小型企業的行銷負責人（25-40 歲）
課程形式：預錄影片課程
預計上架平台：先在自己的網站銷售，之後考慮上架 Hahow
預計時程：希望 2 個月內完成課程製作
可用資源：只有我一個人，預算約 3 萬元

請幫我制定完整的課程企劃，包含：
1. 課程大綱（單元規劃）
2. 製作時程表
3. 所需設備和工具清單
4. 定價策略建議
5. 上架前的宣傳計畫
6. 前 7 天的立即行動清單
```

**AI 的回應（摘要）：**

> **課程大綱（6 單元，約 4 小時）**
>
> | 單元 | 主題 | 預估時長 |
> |------|------|---------|
> | 1 | AI 社群行銷概論：為什麼你需要 AI？ | 30 分鐘 |
> | 2 | 內容創作加速：用 AI 寫出高互動貼文 | 45 分鐘 |
> | 3 | 視覺設計自動化：AI 配圖與設計工具 | 40 分鐘 |
> | 4 | 數據分析與洞察：讓 AI 幫你看懂數字 | 40 分鐘 |
> | 5 | 自動化工作流：從創作到發佈的一條龍 | 45 分鐘 |
> | 6 | 實戰案例：一個月的 AI 社群行銷計畫 | 40 分鐘 |
>
> **製作時程表（8 週）**
>
> | 週次 | 任務 |
> |------|------|
> | 第 1 週 | 完成課程大綱細節、準備講稿 |
> | 第 2-3 週 | 錄製單元 1-3 |
> | 第 4-5 週 | 錄製單元 4-6 |
> | 第 6 週 | 影片剪輯與後製 |
> | 第 7 週 | 製作附贈教材、測試課程平台 |
> | 第 8 週 | 宣傳預熱、正式上架 |
>
> **定價策略建議**
> - 早鳥價：NT$ 1,680（前 100 名）
> - 正式價：NT$ 2,480
> - 定價依據：對標市場同類課程（NT$ 1,500-3,600 區間），以中間偏低切入建立口碑
>
> **前 7 天行動清單**
> 1. Day 1-2：細化課程大綱，每個單元列出具體教學點
> 2. Day 3：購買/準備錄影設備（麥克風、螢幕錄製軟體）
> 3. Day 4-5：撰寫第一單元的完整講稿
> 4. Day 6：試錄 10 分鐘，測試畫質和音質
> 5. Day 7：在社群上發起「你最想學什麼 AI 行銷技巧？」的問卷調查

**你的審核與調整：**

你覺得 8 週的時程有點趕，特別是你還有其他接案工作。你跟企劃 Agent 討論後，把時程調整為 10 週，並在第 6-7 週安排緩衝時間。定價方面，你決定增加一個「VIP 方案」（含一次 1 對 1 諮詢），定價 NT$ 4,980。

### 第三步：用決策輔助 Agent 比較上架平台

最後一個決策點：課程要在哪裡上架？

**你的提示詞：**

```
我正在決定線上課程的上架方式，有三個選項：

選項 A：自架網站（用 Teachable 或 Podia）
選項 B：上架 Hahow
選項 C：先自架，3 個月後再上架 Hahow

我的情況：
- 目前社群粉絲約 3,000 人
- Email 名單約 500 人
- 沒有線上課程的銷售經驗
- 希望長期經營課程事業

評估標準和權重：
1. 初期銷售量：30%
2. 利潤率：25%
3. 品牌建立：20%
4. 學習成本：15%
5. 長期發展性：10%

請用加權評分矩陣分析，並給出你的顧問觀點。
```

**AI 的回應（摘要）：**

> **加權評分矩陣**
>
> | 評估標準（權重）| 選項 A 自架 | 選項 B Hahow | 選項 C 混合 |
> |----------------|------------|-------------|------------|
> | 初期銷售量（30%）| 4 分 | 8 分 | 6 分 |
> | 利潤率（25%）| 9 分 | 5 分 | 7 分 |
> | 品牌建立（20%）| 8 分 | 5 分 | 7 分 |
> | 學習成本（15%）| 5 分 | 7 分 | 5 分 |
> | 長期發展性（10%）| 8 分 | 6 分 | 8 分 |
> | **加權總分** | **6.55** | **6.35** | **6.50** |
>
> **分析說明**
> - **選項 A** 在利潤率和品牌建立上最好，但以你目前 3,000 粉絲的基礎，初期銷售會比較辛苦
> - **選項 B** 能借助平台流量快速獲得銷量，但利潤被平台抽成（通常 30-50%），且品牌建立受限
> - **選項 C** 是折衷方案，先自架建立品牌和收集回饋，再上平台放大聲量
>
> **顧問觀點**
> 考慮你沒有線上課程銷售經驗，我建議選項 C（混合策略），但做一個調整：先在自架平台以「限量早鳥」方式賣給你現有的 500 位 Email 訂閱者，收集回饋、優化課程，然後再上 Hahow 面向更大的市場。這樣你既能保有利潤，又能逐步學習。

**你的最終決策：**

你被選項 C 的混合策略說服了。但你補充一個 AI 沒提到的因素：你有幾位行銷圈的朋友願意幫你推薦課程。有了這個「推薦行銷」的資源，你對自架平台的初期銷售更有信心。

最終決定：選項 C（混合策略），先自架平台銷售。

## 本章重點回顧

1. **市場分析 Agent** 是你的市場研究員，能快速提供 SWOT 分析、競爭分析、趨勢掃描和客群洞察。記得交叉驗證 AI 提供的數據。

2. **企劃 Agent** 是你的專案規劃師，用三步驟工作法（目標釐清 → 結構設計 → 計畫輸出）把想法變成可執行的計畫。

3. **決策輔助 Agent** 是你的策略顧問，提供利弊分析、加權評分、風險評估和情境分析四種框架，幫你做出更理性的決策。

4. 三個 Agent 的協作流程：**分析市場 → 制定計畫 → 比較決策**，形成完整的策略規劃閉環。

5. 記住：AI 是你的智囊團，但最終的指揮棒在你手上。

## 馬上動手

拿出你現在手邊正在煩惱的一個專案或決策，用本章的模板試一次：

1. 用市場分析模板了解你的市場
2. 用企劃模板把想法變成計畫
3. 如果有需要選擇的決策，用決策分析框架攤開來看

你會發現，有了策略 Agent 團隊，「從想法到計畫」的距離比你想像的近得多。
